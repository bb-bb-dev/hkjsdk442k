Compatible Tire Sensors

Compatible tire sensors for TubeWitch

This page groups the tire-pressure sensor families TubeWitch supports today, the sensor family planned next, and the AIRSPY path that depends on custom community firmware.

Current support includes Quarq TyreWiz 1.0 and 2.0 sensors, Quarq TyreWiz for Moto sensors, and Zipp 303 SW wheels with the integrated Zipp AXS Wheel Sensor. Schwalbe AIRMAX is the next sensor family targeted for support.

How to read this page

Quarq / SRAM

These are the Quarq / SRAM sensor families TubeWitch supports today.

Quarq / SRAM

Quarq TyreWiz 1.0

Supported now

The original TyreWiz valve-mounted sensor family remains supported in TubeWitch for live pressure, alerts, and activity recording.

TubeWitch uses only the numeric portion of the printed sensor ID.

Official TyreWiz series

Quarq / SRAM

Quarq TyreWiz 2.0

Supported now

The current TyreWiz generation is supported by TubeWitch for live pressure, stale-data markers, alerts, and Garmin activity recording.

Setup still uses the five-digit numeric sensor ID shown on the hardware.

TyreWiz 2.0 product page

Quarq / SRAM

Quarq TyreWiz for Moto

Supported now

TyreWiz for Moto sensors are also supported by TubeWitch and follow the same core setup flow used by the other TyreWiz families.

TubeWitch still uses the numeric part of the sensor ID only.

TyreWiz for Moto details

Zipp

TubeWitch also supports Zipp 303 SW wheels that use the integrated Zipp AXS Wheel Sensor.

Zipp

Zipp 303 SW with integrated Zipp AXS Wheel Sensor

Supported now

This is the supported Zipp path today. TubeWitch supports the 303 SW wheel system that uses the integrated Zipp AXS Wheel Sensor.

This is not a separate valve-mounted TyreWiz sensor. It is the integrated wheel-sensor path built into the 303 SW system.

303 SW product page AXS Wheel Sensor details

Support notes

What this support means

  • TubeWitch support applies to 303 SW wheels that use the integrated Zipp AXS Wheel Sensor.
  • Once the wheel system is configured in SRAM AXS, TubeWitch uses the normal live-pressure and alert workflow.
  • The supported hardware path is the wheel system itself, not a separate aftermarket sensor.

Schwalbe

AIRMAX is the next smart-sensor family targeted for TubeWitch compatibility.

Schwalbe

Schwalbe AIRMAX Smart Bike Sensor

Coming soon

TubeWitch support for AIRMAX is planned for a future update. AIRMAX uses paired front and rear sensors with live pressure monitoring through the Schwalbe app.

This is the next sensor family targeted after the current Quarq / SRAM and Zipp support already in TubeWitch.

Official AIRMAX page

AIRMAX app

Pairing and setup style

The AIRMAX workflow uses a dedicated app for pairing, front and rear assignment, and pressure targets. That makes it a strong fit for future TubeWitch-style support.

SKS

AIRSPY only makes sense for TubeWitch through the community firmware path.

AIRSPY MUST be running the custom firmware by bitmeal.

The stock AIRSPY product is not the TubeWitch target here. The relevant path is the community firmware project that exposes AIRSPY through the custom ANT+ implementation.

github.com/bitmeal/sks_airspy_ant_community_fw

SKS

SKS AIRSPY

Custom firmware required

AIRSPY hardware is relevant to TubeWitch only through the bitmeal community firmware path. That firmware requirement is the critical detail, not the stock AIRSPY software stack.

If AIRSPY is part of your plan, treat the bitmeal firmware as mandatory before expecting TubeWitch compatibility.

Official AIRSPY page bitmeal firmware project

Important

Do not treat stock AIRSPY as enough

The hardware alone is not the compatibility answer.

For this sensor family, the firmware path matters first. If the custom firmware is not in place, this is not the right TubeWitch route.

TubeWitch compatibility depends on the exact tire-sensor family and, in some cases, the exact firmware path.

Use this page to separate supported hardware, planned hardware, and firmware-dependent hardware before setting up a bike.
